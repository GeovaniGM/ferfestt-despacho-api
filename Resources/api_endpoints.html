API — Sistema de Despacho de Pedidos

Base URL: https://api.empresa.com/api/v1  ·  Versão 1.0  ·  Última atualização: março/2026

Autenticação
POST /auth/login pública
Autenticação: Pública

Autentica o usuário administrador e retorna um token JWT com validade de 8 horas.

Request body
{
  "email": "admin@empresa.com",
  "senha": "senha-segura"
}
Response 200
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expira_em": "2026-03-31T23:00:00.000Z"
}
Response 401
{
  "erro": "CREDENCIAIS_INVALIDAS",
  "mensagem": "Email ou senha incorretos."
}
App do operador
GET /bancadas/:id Validar bancada app
Autenticação: API Key — X-Api-Key

Chamado quando o operador lê o código de barras da bancada. Valida se existe e está ativa. O app salva o resultado localmente para uso offline.

Response 200
{
  "id": "uuid-da-bancada",
  "codigo": "BANCADA-07",
  "descricao": "Bancada expedição leste",
  "ativo": true
}
Response 404
{
  "erro": "BANCADA_NAO_ENCONTRADA",
  "mensagem": "Código de bancada não reconhecido."
}
POST /despachos Registrar despacho app
Autenticação: API Key — X-Api-Key

Registra um único despacho. O backend valida o DV da chave de acesso, checa duplicidade e persiste o registro. Um evento é gravado em despacho_eventos independente do resultado.

Request body
{
  "chave_nfe": "41260353067994000120550010000901741254185566",
  "bancada_id": "uuid-da-bancada",
  "lido_em": "2026-03-31T14:32:10.000Z"
}
Response 201 — criado
{
  "id": "uuid-do-despacho",
  "chave_nfe": "41260353067994000120550010000901741254185566",
  "numero_nfe": "000090174",
  "bancada_id": "uuid-da-bancada",
  "lido_em": "2026-03-31T14:32:10.000Z",
  "registrado_em": "2026-03-31T14:32:11.432Z"
}
Response 409 — duplicidade
{
  "erro": "CHAVE_DUPLICADA",
  "mensagem": "Esta NFe já foi despachada.",
  "despacho_id": "uuid-do-registro-original"
}
Response 422 — dados inválidos
{
  "erro": "CHAVE_INVALIDA",
  "mensagem": "Chave de acesso inválida."
}
POST /despachos/lote Sincronizar fila offline app
Autenticação: API Key — X-Api-Key

Usado quando o app retorna à conectividade e precisa sincronizar múltiplos registros acumulados. Cada item é processado individualmente — um erro não cancela os demais. Retorna HTTP 207 Multi-Status.

Request body
{
  "bancada_id": "uuid-da-bancada",
  "registros": [
    {
      "chave_nfe": "41260353067994000120550010000901741254185566",
      "lido_em": "2026-03-31T08:10:00.000Z"
    },
    {
      "chave_nfe": "41260353067994000120550010000901741254185567",
      "lido_em": "2026-03-31T08:12:44.000Z"
    }
  ]
}
Response 207 — multi-status
{
  "resultados": [
    {
      "chave_nfe": "41260353067994000120550010000901741254185566",
      "status": "criado",
      "despacho_id": "uuid"
    },
    {
      "chave_nfe": "41260353067994000120550010000901741254185567",
      "status": "rejeitado",
      "erro": "CHAVE_DUPLICADA"
    }
  ]
}

DELETE /despachos/:id Soft delete admin
Autenticação: JWT — Authorization: Bearer

Define ativo = false no despacho e registra um evento do tipo inativado em despacho_eventos com o motivo informado. Libera a chave NFe para novo despacho.

Request body
{
  "motivo": "NFe lida incorretamente pelo operador"
}
Response 200
{
  "id": "uuid",
  "ativo": false,
  "evento": {
    "id": "uuid-do-evento",
    "tipo": "inativado",
    "motivo": "NFe lida incorretamente pelo operador",
    "criado_em": "2026-03-31T15:00:00.000Z"
  }
}
Response 404
{
  "erro": "DESPACHO_NAO_ENCONTRADO"
}
